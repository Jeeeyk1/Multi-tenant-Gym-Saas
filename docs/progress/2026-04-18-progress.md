## Decisions made

### Issue 1 — Check-in outside schedule
**Decision:** Flag behavior, not hard block.
Added `check_ins.is_out_of_hours boolean NOT NULL DEFAULT false`.
Out-of-hours check-ins proceed and are flagged for staff review.

### Issue 2 — Suspension scope
**Decision:** Gym-scoped suspension.
`gym_members.status = 'SUSPENDED'` blocks gym feature access only.
Login is unaffected. Only `users.is_active = false` blocks login globally.

### Issue 3 — Login architecture
**Decision:** Two-tier login model.
- Org owners log in via org slug → org-level JWT (`organizationId`, `orgRole`)
- Staff and members log in via gym code → gym-level JWT (`gymId`, `roles`, `permissions`)
- Shared `POST /auth/resolve-code` endpoint detects which type of code was entered
- One account per gym. Same person at two gyms = two separate accounts, two different emails.
- Both web and mobile support org-level and gym-level users.

### Issue 4 — Auto-suspend rule
**Decision:** Auto-suspend after 3 months of EXPIRED status.
Config stored per gym: `gyms.auto_suspend_months INT NOT NULL DEFAULT 3`.
Handled by `AutoSuspendJob` cron (daily at 02:00 UTC).

### Issue 5 — Gym code format
**Decision:** 12-character uppercase alphanumeric, auto-generated from gym name, not changeable.
Algorithm: strip vowels from name → take first 7 chars → append random suffix to reach 12.
Safe character set excludes ambiguous chars (0, O, 1, I, L).

### Issue 6 — Check-out behavior
**Decision:** Auto-checkout via background cron + runtime safety net.
- `AutoCheckoutJob` runs every 15 minutes, closes stale check-ins past `gyms.auto_checkout_hours` (default: 5)
- At check-in time, if member has an open stale session, it is auto-closed before the new check-in is created
- Added `check_ins.is_auto_checkout boolean NOT NULL DEFAULT false`

### Issue 7 — `updated_at` strategy
**Decision:** Application-level. Every `update()` call explicitly sets `updatedAt: new Date()`.
No DB triggers.

### Issue 8 — MVP scope
**Included:** Organizations, gyms, staff RBAC, plans, members, check-ins, renewals, announcements, community chat, all 4 cron jobs.
**Excluded from MVP:** AI assistant, plan_promos logic (schema defined only), advanced reporting, direct/group chat.
