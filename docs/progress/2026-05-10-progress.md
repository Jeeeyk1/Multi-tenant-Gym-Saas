## Phase worked on

## What was completed

### Bug 1: Member dashboard — "Missing permission" + wrong ID type

**Root cause (two problems):**
- `getMember(gymId, user.id)` passed the USER ID as `memberId`, but `GET /gyms/:gymId/members/:memberId` expects the GYM MEMBER record ID.
- The endpoint requires `members.view` permission which members never have.

**Fixes:**
- Added `findFullMemberByUserId(gymId, userId)` to `MembersRepository` — looks up by user ID and returns full member data including plan.
- Created `GetMyMemberUseCase` — no `members.view` required; maps `membershipPlan` → `plan` to match the mobile `GymMember` type.
- Added `GET /gyms/:gymId/members/me` to `MembersController` (declared before `:memberId` to avoid route conflict).
- Registered `GetMyMemberUseCase` in `MembersModule`.
- Mobile: replaced `memberService.getMember(gymId, userId)` with `memberService.getMyMember(gymId)`.

### Bug 2: Check-in history on dashboard — "Missing permission"

**Root cause:**
- `getCheckIns()` called `GET /gyms/:gymId/checkins` which requires `checkins.view` — members only have `checkins.self`.
- The `memberId` query param was also unsupported by the DTO.

**Fixes:**
- Added `listCheckinsByUserId(gymId, userId, limit)` to `CheckInsRepository`.
- Created `ListMyCheckInsUseCase` — requires `checkins.self`; filters by `caller.sub`.
- Added `GET /gyms/:gymId/checkins/me` to `CheckInsController` (declared before `active` and `/:checkinId`).
- Registered `ListMyCheckInsUseCase` in `CheckInsModule`.
- Mobile: replaced `memberService.getCheckIns(gymId, userId, limit)` with `memberService.getMyCheckIns(gymId, limit)`.
- Mobile: simplified `selfCheckIn(gymId)` — no longer passes `memberId` (use case uses `caller.sub` internally for `APP_SELF_CHECKIN`).

### Bug 3: Chat — WebSocket namespace mismatch

**Root cause:**
- `io(\`${API_URL}/chat\`)` where `API_URL = http://host:3000/api/v1` → tries to join namespace `/api/v1/chat`, but the NestJS gateway is at namespace `/chat`.

**Fix:**
- Derive `WS_BASE_URL` by extracting just the origin from `EXPO_PUBLIC_API_URL` using `new URL(apiUrl).origin`.
- Socket now connects to `http://host:3000/chat` → correct namespace `/chat`.

## Key decisions

- Self-access endpoints (`/me`) are the right pattern: they avoid permission escalation while giving members access to their own data. Using `caller.sub` (user ID) as the lookup key is safe since it comes from the verified JWT.
- The `/members/me` endpoint maps `membershipPlan` → `plan` in the response to match the mobile `GymMember` type rather than updating the mobile type.

## Second round of fixes (same session — v0 pattern cleanup)

### Sign out not working
**Root cause:** After `signOut()` sets `user = null`, the app stays on the member screen because the Stack doesn't re-evaluate. The member layout had no auth guard.

**Fix:** Added `useAuth()` to `(member)/_layout.tsx`. When `!isAuthenticated && !isLoading`, it renders `<Redirect href="/(auth)/entry" />`. This protects ALL member routes and handles sign-out navigation automatically.

### Chat socket lifecycle race condition
**Root cause:** Socket was created inside an async `useCallback` chained through `useEffect`. This created timing issues: if the async call was in-flight when the component unmounted, the socket could connect after cleanup ran, leaving it dangling.

**Fix:** Rewrote chat screen with a single `useEffect` using a `mounted` flag. The flag guards every async step — if the component unmounts mid-setup, no state is set and no socket is kept. Cleanup is now `socket?.disconnect()` called synchronously in the effect return.

### Check-in error code mismatch
**Root cause:** Check-in screen handled `MEMBERSHIP_SUSPENDED` and `GYM_CLOSED` — neither code exists in the API. The API throws `MEMBER_NOT_ACTIVE` (for inactive/suspended members) and `MEMBER_NOT_FOUND`.

**Fix:** Updated error code handling to match real API: `MEMBER_NOT_ACTIVE`, `MEMBER_NOT_FOUND`, `ALREADY_CHECKED_IN`, `MEMBERSHIP_EXPIRED`.

### Dev seed member expiry date stale
**Root cause:** Seed used `ON CONFLICT DO NOTHING` so the member's expiry date (set to `NOW() + 30 days` on first seed run) never updated. After 30 days, all check-ins fail with `MEMBERSHIP_EXPIRED`.

**Fix:** Changed seed to `ON CONFLICT (id) DO UPDATE SET status = 'ACTIVE', expiry_date = NOW() + INTERVAL '365 days'`. Re-running the seed now refreshes the dev member.

## Third round of fixes — Web app foundation rewrite

**Context:** The web app scaffold (`gym-setup-with-shadcn/`) was generated by v0.vercel.ai and used entirely invented API contracts (wrong endpoints, wrong field names, single-token auth, no WebSocket). The "Not a conversation member" error on chat was a symptom of this.

### Root causes identified
1. `.env.local` used `http://localhost:3001/api` (missing `/v1`)
2. Auth layer called `/auth/me` which doesn't exist — API has no "current user" endpoint
3. Login stored single `token` but API returns `{ accessToken, refreshToken, user: { id, email, fullName } }`
4. JWT payload `GymAuthUser` contains `{ sub, type, gymId, roles[], permissions[] }` — no `email`, `fullName`, `gymCode`
5. Chat service called `/chat/conversations/...` instead of `/gyms/:gymId/conversations/...`
6. Chat page used polling (setInterval) instead of WebSocket
7. `User` type had v0 fields (`firstName`, `lastName`, `role` string) instead of real fields (`fullName`, `roles` array)

## Key architecture decisions (web)

- **No `/auth/me` endpoint** — session is restored from localStorage only; JWT is self-contained
- **Two-token auth**: `accessToken` in Authorization header; `refreshToken` sent to `POST /auth/refresh`; both stored in localStorage
- **JWT decode**: `atob(token.split('.')[1])` — no jwt-decode library needed
- **WebSocket URL**: derived as `new URL(NEXT_PUBLIC_API_URL).origin` to strip `/api/v1` path prefix
- **`AuthUser` type**: `{ id, email, fullName, gymId, roles[], permissions[] }` — combines login response body + JWT payload
