## Key decisions

- **Onboarding route is outside `(member)` tabs** — full-screen flow, no tab bar. Placed at `apps/mobile/src/app/onboarding.tsx`.
- **All steps are skippable except activity level** — final step's selection triggers submission immediately for a smoother flow. "Skip for now" on the last step submits with empty `activityLevel`.
- **`onboardingDone` check happens in `index.tsx`** — keeps the tab layout clean; the check is a lightweight API call made once per session.
- **QR code uses `qrCodeToken` directly** — the token is already a unique 64-char hex string; no additional encoding needed.
- **`findMyProfile` returns a stub if no profile row exists yet** — the API use case returns `{ memberId, onboardingDone: false }` so the mobile client always gets a valid response.
