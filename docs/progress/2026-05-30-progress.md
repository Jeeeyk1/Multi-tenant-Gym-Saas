## Key decisions
- **`warning` ‚Üí burn orange**: `#FF6B35` replaces amber across warning states (suspended badge, already-checked-in, streak chip). Orange communicates urgency without breaking meaning.
- **Check-in card text dark**: the mint‚Üíblue gradient starts light, so dark text is necessary for contrast on the card. Success/error button states (green, red) retain white text.
- **Onboarding `btnPrimary` restructured**: split into `btnPrimary` (wrapper with borderRadius + overflow) and `btnGradient` (LinearGradient inner). Opacity disabled state applied to the wrapper.
- **`GRADIENTS` export added to theme**: provides typed gradient color tuples so screens don't inline hex arrays.

### Animated splash screen (`frontend-2.md`)

**New file:** `apps/mobile/src/components/AppSplash.tsx`
- Background `#0F0F0F`, centered layout
- Mascot (üêî) springs in from scale 0.5 ‚Üí 1.0 with bounce ‚Äî `damping: 10`
- "GainzOS" app name fades + slides up at 400ms delay
- "Train. Connect. Grow." appears letter by letter (38ms stagger per character) starting at 800ms
- Mint green glow ring pulses once around the logo at 600ms (scale 1 ‚Üí 1.7, opacity 0 ‚Üí 0.55 ‚Üí 0)
- 2px mint progress bar at bottom animates 0 ‚Üí 100% over 2000ms
- Entire screen fades out at 2300ms, calls `onDone` when done

**Updated:** `apps/mobile/src/app/_layout.tsx`
- After fonts load, renders `<AppSplash>` instead of the navigator
- `splashDone` state gates the transition ‚Äî splash shows once per app session
- `AuthProvider` is deferred until after splash (avoids auth loading spinner overlapping)

**Key decisions:**
- Spec mentions Framer Motion ‚Äî not applicable to React Native; used `Animated` API (already the project standard)
- `useNativeDriver: false` only on the progress bar (width is a layout prop); all other animations run on native thread
- Splash is component-level state in `RootLayout`, not a separate route ‚Äî no routing edge cases
