## Key Decisions
- **Tailwind v3 over v4**: shadcn/ui compatibility; CSS variable approach (`hsl(var(--primary))`) is well-supported
- **Server Actions for auth**: `login()` sets httpOnly cookies server-side; tokens never touch client JS
- **`useState` not `useTransition` for async**: React 18 doesn't support async callbacks in `startTransition`; used `useState(false)` for loading state
- **`resolveCode` + `?type=` param**: login page receives the token type (gym/org) from the URL so it can call the correct login endpoint
