## Key Decisions

- **Epley formula stored at submission time**: `estimated_1rm` is computed once on submit and persisted, not recalculated on read. This keeps leaderboard queries simple and ensures historical consistency if the formula ever changes.
- **`UNIQUE NULLS NOT DISTINCT`**: PostgreSQL 15+ syntax needed because standard `UNIQUE` treats NULLs as distinct values ‚Äî without it, multiple global exercises with the same name could be inserted.
- **Two named User relations on `MemberPrSubmission`**: Prisma requires named relations when a model has two foreign keys to the same model (`submittedByUserId` ‚Üí `"SubmittedBy"`, `reviewedByUserId` ‚Üí `"ReviewedBy"`).
- **`groupBy` + `findFirst` pattern**: Prisma's `groupBy` returns the best 1RM per member but not full submission details. A follow-up `findFirst` per result fetches the complete row.
- **Mobile multipart upload**: React Native `fetch` + `FormData` requires omitting `Content-Type` ‚Äî fetch sets the multipart boundary automatically. File is represented as `{ uri, name, type }` object appended to `FormData`.
- **Staff-submitted PRs auto-approved**: When `memberId` is present in `SubmitPrDto`, the submission type is STAFF and status is set to APPROVED immediately ‚Äî no review step needed.
