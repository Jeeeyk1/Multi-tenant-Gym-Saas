## Key Decisions

- **Plans live fetch with fallback**: `PricingSection` is a client component for the billing toggle; the page fetches plans server-side and passes them as props. When the API is unavailable the component renders three hardcoded demo plans so the page always looks good.
- **Revalidate 60s**: Plans are cached for 60 seconds with ISR so the page stays fast and picks up admin changes without redeployment.
- **No landing app in monorepo turbo `dev` default**: `dev:landing` is a separate script so the landing page doesn't run in daily API+web development.
