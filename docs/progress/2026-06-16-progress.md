## Key Decisions

- **No HealthKit in Phase 1** — keeps everything Expo Go testable. HealthKit needs a native build (`npx expo run:ios`).
- **Timer is JS `setInterval`** — sufficient for Phase 1; HealthKit will provide accurate timestamps in Phase 2.
- **Idle detection is time-based** — 25 min in-app timer + `expo-notifications` for background. Simpler than motion/accelerometer.
- **API save is non-blocking** — workout summary is shown immediately; if save fails, the user still sees their session data.
- **Token type backward compat** — `findInviteToken` accepts both `'INVITE'` and `'MEMBER_INVITE'` to handle tokens already in the DB.
