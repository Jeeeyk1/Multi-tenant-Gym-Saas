### Key decisions

- **Cycle badges use `badge_catalog` not dynamic `GymCustomBadge`**: avoids the `created_by_user_id NOT NULL` constraint issue on auto-generated records; `proof_notes` stores the exercise name for UI display context
- **Fire-and-forget badge checks**: badge award failures must never surface to the caller (session log or check-in); errors are logged only
- **`BadgesService` facade**: keeps cross-module coupling minimal — consumers call two methods without knowing the internal use-case structure
- **`is_auto_awarded = FALSE` for cycle catalog entries**: prevents the generic `CheckAutoBadgesUseCase` from picking them up; only `CloseCycleUseCase` awards them
