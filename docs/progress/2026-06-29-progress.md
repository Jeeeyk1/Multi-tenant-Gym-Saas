## 1. Exercise media — GIF dataset migration

### What changed
- `db/seeds/seed-exercises.ts` rewritten to pull from `hasaneyldrm/exercises-dataset` instead of `yuhonas/free-exercise-db`.
- New dataset has actual GIF animations (`videos/*.gif`) rather than static JPEGs.
- GIF-first with JPEG fallback: if `gif_url` is missing for any exercise, the seed stores the `image` (JPEG) URL so no exercises are silently dropped.
- Default mode is now **truncate + re-seed** (clean run). Pass `--upsert` to skip the truncate.
- Field mapping: `target` → `primaryMuscles`, `category`/`body_part` → `category`.

### Name-mapping notes
The AI workout prompt already constrains the model to "use ONLY these exact names (match case exactly as shown)" from the `exercise_media` table. After re-seeding, future AI workout plans automatically use the new dataset's names. Old saved plans may get `null` from `GetExerciseMediaUseCase` but the graceful null-return is already in place.

### How to apply
```bash
pnpm seed:exercises    # truncates exercise_media and re-seeds from new dataset
```

## 2. Mobile UI refresh — light/dark mode + layout improvements

### What changed

#### Color system foundation
- `apps/mobile/src/constants/theme.ts` — added `DARK_COLORS` and `LIGHT_COLORS` as named exports. `COLORS` kept as `DARK_COLORS` alias for any un-migrated code.
- `apps/mobile/src/context/ThemeContext.tsx` — extended `GymTheme` with `colorScheme: 'light' | 'dark'` and `colors: ColorPalette` (the full mode-sensitive color set). Added `toggleColorScheme()` to context; persists preference in AsyncStorage under `gym_color_scheme`. The tenant's `primary`/`secondary` are unaffected.

#### Shared components
- `GreetingBanner` — replaced AnimatedMascot with an initials avatar circle (prop: `initials`). Uses `theme.colors` for text. Added streak chip with proper light-mode background.
- `MembershipCard` — uses `theme.colors` via `useMemo`-wrapped StyleSheet.

#### Tab bar (`(member)/_layout.tsx`)
- Tab bar `backgroundColor`, `borderTopColor`, `tabBarInactiveTintColor` now pulled from `theme.colors` (responds to light/dark).
- Added `whos-here` and `membership` to hidden routes (they were navigable but not declared).

#### Secondary screens — color system only
`whos-here.tsx`, `membership.tsx`, `announcements.tsx`, `leaderboard.tsx`, `workout-history.tsx` all migrated to `useMemo`-wrapped StyleSheet with `theme.colors`.

#### Still on static `COLORS` (dark mode always)
`ai.tsx`, `chat.tsx`, `workout.tsx` — complex screens deferred for a follow-up pass. They render fine in dark mode (COLORS = DARK_COLORS). They won't respond to the light mode toggle until updated.

### Key design decisions
- **Pattern: `useMemo(() => StyleSheet.create({...}), [C])`** — standard RN theming pattern. StyleSheet is only recreated when color scheme changes (rare). No performance concern.
- **Tenant primary color unchanged** — `theme.primary` / `theme.secondary` / `theme.gradient` remain fully tenant-configurable, separate from the light/dark system.
- **Equal 5-tab bar** — no raised center button. All tabs have equal visual weight; Check In stays center by position, not by prominence.
