# Progress — 2026-07-02

## Session: Admin AI Insights Assistant — Phase 3 (web UI) + Phase 4 (mobile UI)

Both client surfaces for the admin AI assistant are now built on top of the Phase 1/2 API
(see 2026-07-01 progress for the API core). Non-streaming request/response; the client holds
conversation history and sends it on each turn (stateless server).

### Phase 3 — Web dashboard chat

- `apps/web/src/types/api.ts` — added `InsightsResult { answer, toolsUsed }`.
- `apps/web/src/lib/actions/insights.ts` — `queryInsights(gymId, message, history)` server action
  wrapping `POST /gyms/:gymId/insights/query` (mirrors the existing action pattern).
- `apps/web/src/components/insights/chat-client.tsx` — client chat UI: scrollable message list +
  sticky input, empty state with 4 tappable suggestions, user/assistant bubbles, animated
  thinking dots, a "✦ Queried metrics, trend data" transparency line per answer, error bubbles,
  Enter-to-send / Shift+Enter newline. History accumulates in React state.
- `apps/web/src/app/dashboard/insights/page.tsx` — server component, redirects to `/dashboard`
  if the caller lacks `insights.query`. Uses `min-h-0` flex so the chat scrolls within the fixed
  dashboard layout (`h-screen overflow-hidden`).
- `apps/web/src/components/layout/sidebar.tsx` — "AI Assistant" nav item (✦), gated by
  `insights.query` so only MANAGER/GYM_OWNER see it.

Verified: `tsc --noEmit` 0 errors; `next lint` clean (only the pre-existing unrelated `<img>`
warning on the login page).
